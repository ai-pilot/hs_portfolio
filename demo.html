Detection Pipeline
📄
Raw Invoices
ERP + Scanned
Ingest invoices from ERP (SAP, Oracle) and scanned documents via Azure Document Intelligence. Handles PDF, images, and structured data.
➔
🔬
Normalize
OCR-Aware
Canonical normalization strips suffixes (CANCEL, REV, DUP), resolves OCR confusables (O↔0, I↔1, B↔8), and extracts numeric runs for fuzzy matching.
➔
🗃
Blocking
5 Index Types
Candidate generation via 5 blocking indexes: normalized number, digit-only, canonical base, numeric runs, and vendor+amount bucket. Reduces O(n²) to O(n·k).
➔
📊
Scoring
Multi-Signal
Each candidate pair scored on: invoice number similarity (Damerau-Levenshtein + OCR-aware), amount match (incl. FX tolerance), vendor cluster, description overlap, date proximity.
➔
🧠
AI Review
LLM + Guardrails
High-risk pairs reviewed by GPT-4 with forensic audit prompt. Prompt injection protection via sanitization. Response validated against strict schema. Rate-limited with audit trail.
➔
🎯
Priority Queue
P1-P5 Routing
Final priority (P1=critical to P5=dismiss) computed from AI verdict, confidence, risk score, and escalation categories. Cross-vendor fraud always routes to P1.
